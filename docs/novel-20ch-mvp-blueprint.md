# 20 章 AI 小说 MVP 工作流蓝图

## 目标

第一轮只做 20 章，目标是验证“从标题开始，到章节归档、摘要回写、质量审查”的完整闭环。

示例标题：`我要送快递可是你非要我做董事长`

这个阶段不追求全自动百万字，也不追求复杂 RAG。重点是把流程跑通，把中间产物保存下来，验证 AI 能否持续 20 章不跑偏。

## 标题解析

标题本身已经包含故事张力：

- 表层身份：主角想送快递，代表底层、自由、普通生活。
- 外部强迫：别人非要他做董事长，代表权力、责任、误会、继承、阴谋。
- 核心反差：主角越想低调，外界越把他推到高位。
- 读者期待：轻喜剧、都市爽文、反套路继承、商业打脸、身份错位。

GPT 在第一步要把标题扩展成故事种子，而不是直接写正文。

## 20 章 MVP 总流程

```text
标题输入
  -> 故事种子生成
  -> 小说圣经生成
  -> 20 章大纲生成
  -> 按章节循环
      -> 章节任务单
      -> 3-5 个场景卡
      -> Kimi 逐场景扩写
      -> 合并成章
      -> GPT 审稿评分
      -> 不合格则重写或修订
      -> 归档章节 Markdown
      -> 摘要/角色状态/伏笔状态回写
  -> 20 章合订归档
```

## 推荐 N8N 子流程

### 01_seed_from_title

输入：

- `title`
- 可选：题材、目标读者、文风、禁忌、章节数，默认 20。

输出：

- `book_id`
- `genre`
- `reader_promise`
- `core_hook`
- `protagonist`
- `main_conflict`
- `tone`
- `ending_direction`

### 02_story_bible

输入：故事种子。

输出：

- 世界背景
- 主角设定
- 主要配角
- 反派/阻力系统
- 公司/势力设定
- 爽点机制
- 喜剧机制
- 伏笔池
- 文风规则
- 禁止事项

### 03_outline_20

输入：小说圣经。

输出：20 章章节表。每章至少包含：

- `chapter_index`
- `chapter_title`
- `chapter_goal`
- `main_conflict`
- `key_reveal`
- `emotional_beat`
- `state_changes`
- `cliffhanger`

### 04_chapter_pipeline

输入：单章计划 + 当前记忆。

步骤：

1. GPT 生成章节任务单。
2. GPT 拆成 3-5 个场景卡。
3. Kimi 逐场景扩写。
4. Kimi 或 GPT 合并润色。
5. GPT 审稿评分。
6. 低分重写，高风险转人工。
7. 保存正文。

### 05_memory_update

输入：最终章节正文。

输出：

- 本章 300 字摘要
- 新增事实
- 角色状态变化
- 关系变化
- 伏笔新增/推进/回收
- 下一章必须记住的事项

### 06_finalize_book

输入：20 章正文和摘要。

输出：

- 全书 Markdown 合订版
- 20 章摘要总表
- 角色最终状态
- 未回收伏笔清单
- 质量报告

## 示例 20 章方向

### 第一幕：身份错位与被迫上位

1. 主角送快递进错会议室，被所有董事误认为神秘继承人。
2. 主角试图解释身份，却意外解决公司物流危机。
3. 董事会认定他是低调考察基层的少东家。
4. 真正的继承文件出现，主角名字竟然真的在名单里。
5. 主角坚持只想送快递，却被迫接手濒临破产的子公司。

### 第二幕：低层经验反杀商业套路

6. 主角用快递路线经验重构仓配系统。
7. 反派高管设局让他签失败项目。
8. 主角靠一线人脉发现供应链猫腻。
9. 女主/关键搭档开始怀疑他不是草包。
10. 子公司首次盈利，主角却更想辞职。

### 第三幕：身份真相与权力陷阱

11. 主角发现父亲/养父与集团创始人有关。
12. 反派制造舆论，说主角学历造假、履历造假。
13. 主角公开承认自己确实是快递员，反而赢得基层支持。
14. 董事会逼宫，要求他交出管理权。
15. 主角用真实业务数据反击，拿到关键投票。

### 第四幕：主动选择与第一阶段胜利

16. 主角发现集团最大危机不是钱，而是内部系统性腐败。
17. 反派绑架/威胁关键证人或关键数据。
18. 主角用快递网络完成一次反套路取证。
19. 董事长身份终于坐实，但主角提出自己的规则。
20. 第一阶段收束：他不再逃避董事长身份，但保留快递员视角，开启下一卷。

## 质量闸门

每章 GPT 审稿至少输出：

- `logic_score`
- `character_score`
- `pace_score`
- `hook_score`
- `style_score`
- `problems`
- `rewrite_instruction`

推荐阈值：

- 总分低于 80：自动修订一次。
- 总分低于 70：整章重写。
- 出现重大设定冲突：暂停人工确认。

## 第一轮不做的内容

- 不做复杂向量数据库。
- 不做全自动百万字。
- 不做多平台发布。
- 不做过多模型路由。
- 不让 AI 自动改核心设定。

第一轮的胜负手是：20 章能不能连续、好读、不吃书，并且每一章都有可追踪的摘要和状态变化。
