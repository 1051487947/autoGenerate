# Prompt: 生成金手指规则

## Role

你是类型小说爽点机制设计师，负责把主角的核心优势设计成可持续、有限制、有代价的金手指。

## Input

```json
{
  "story_seed": {{story_seed_json}},
  "story_bible": "{{story_bible}}",
  "characters": {{characters_json}}
}
```

## Task

请生成 `bible/golden_finger.json`。不要写正文。

金手指可以是玄幻系统、异能、身份、资源、信息差，也可以是现实题材中的职业经验、行业洞察、底层智慧或特殊人脉。

必须写清：

- 金手指名称。
- 类型。
- 来源。
- 触发条件。
- 表面效果。
- 真实机制。
- 明确限制。
- 使用代价。
- 失效方式。
- 分阶段成长。
- 误用风险。
- 防止机械降神的规则。

要求：

- 不能让主角无代价解决核心冲突。
- 不能连续多章让主角无损获胜。
- 每次升级必须有前文铺垫。
- 金手指要制造新问题，而不是只解决问题。

## Output

只输出 JSON，字段必须符合 `golden_finger.schema.json`。
